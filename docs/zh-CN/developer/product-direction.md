# 产品方向与设计原则

英文版：[../../en/developer/product-direction.md](../../en/developer/product-direction.md)

这个文件记录 Culvia 的长期产品判断。它不是面向外部发布的介绍稿，而是后续每轮迭代的取舍依据。

## 调研输入

- [Aftershoot Select](https://aftershoot.com/selects/)：自动初筛帮助摄影师更快复核、覆盖结果，并保留星级、颜色标签、旗标导出到 Lightroom Classic 或 Capture One。
- [Adobe Lightroom Classic 评级/旗标/颜色标签](https://helpx.adobe.com/ca/lightroom-classic/help/flag-label-rate-photos.html)：成熟选片工具把评分、旗标和颜色标签作为浏览、比较、筛选的基础语言。
- [Capture One rating and color tags](https://support.captureone.com/hc/en-us/article_attachments/360007177777)：专业工作流长期依赖星级和颜色标签组织图片，工具应降低决策摩擦，而不是用复杂面板打断审片。
- [Narrative Select](https://narrative.so/)：高频选片强调快速加载、快速判断和对焦/闭眼等可解释信号。

## 产品定位

Culvia 面向摄影师，帮助一组照片走向清晰、可交付的作品选择。它应该像专业照片工具一样尊重摄影师的最终判断，同时把机器评分、大模型评价和技术质检组织成可复核的第二意见。

## 设计原则

- 大图优先：首屏重点永远是当前照片，评分和建议服务于审片，不抢主视觉。
- 模型辅助，摄影师决策：自动评分用于排序、提醒和批量初筛，人工入选/待复核/淘汰始终拥有最高优先级。
- 解释要可扫读：大模型长文必须结构化为整体评价、审美依据、技术问题、修图建议、拍摄建议，而不是整段堆文字。
- 专业但不冰冷：视觉语言以安静、高质感、低干扰为主，颜色用于状态和层级，不做无意义装饰。
- 本地与 Web 同源：Web、本地 app、pip 分发共用同一套 API 和核心评分逻辑；原生能力通过 capabilities 显示和降级。
- 隐私默认收紧：本地模型不上传图片；大模型评审必须显式配置；API key 不进入 SQLite 和 Git。

## 工作流骨架

1. 导入：选择本地目录、拖拽文件夹或上传临时图片。
2. 评分：自动准备模型，按图片展示当前进度、已完成评估和可暂停状态。
3. 初筛：按综合推荐、模型分歧、技术问题、大模型意见缩小候选。
4. 复核：在选片台用大图、缩略图栏、星级、入选/待复核/淘汰完成最终判断。
5. 标记：用星级、旗标状态和颜色标签形成跨工具可理解的专业选片语言。
6. 导出：导出已选照片、CSV 明细，未来支持 Lightroom/Capture One 友好的评级与标签映射。

## 操作历史与撤销原则

- 撤销属于主工作流：误操作后的第一反应应该是 toast 或 `Cmd/Ctrl+Z`，而不是打开设置抽屉找历史。
- 历史属于审计和理解：最近操作面板应解释发生了什么、是否已恢复、是否仍可恢复，但默认不承担高频操作。
- 不覆盖后续判断：如果用户在某次批量操作后又手动改过相关照片，撤销应保护性拒绝，而不是静默回滚。
- 输入优先原生体验：焦点在路径、筛选名、大模型配置或提示词编辑区时，`Cmd/Ctrl+Z` 必须保留为文本撤销。
- 行内撤销要克制：未来如果历史面板支持撤销按钮，应先展示影响范围和冲突风险，避免把抽屉变成第二个选片台。
- 语言要像选片工具：使用“恢复上一步”“已恢复”“可撤销”这类面向摄影工作流的词，不暴露 payload、history id 或数据库细节。

## 后续路线

- 架构：继续拆分 `culvia_app.py`，形成 app factory、state store、评分服务、文件服务、导出服务。
- 前端：把导入、模型、筛选改成任务抽屉和工作台工具条，减少侧栏配置噪声。
- 视觉：统一设计 token 和图表色系，避免每个模块单独发明颜色。
- 本地 app：继续推进桌面路线，复用 `culvia-supervisor` supervisor 和 Starlette API；当前桌面壳实现使用 Tauri，pywebview 只作为轻量备选，Electron 暂不作为默认路线。
- 选片：增加快捷键、批量采纳、颜色标签、重复/相似组比较、导出映射。
- 大模型：保留 OpenAI-compatible 接口，支持模型列表读取、提示词预设、结果版本化和失败重试。
